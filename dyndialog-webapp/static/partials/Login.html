Welcome to this Context Driven Dialog demonstration. The application is built as an example to demonstrate a customer care application for a mobile telecom company.

This application simulates a Customer Support front end which lets a registered user interact with the support system using natural language. As the conversation progresses the support system will load relevant customer data from external systems, such as a Customer Relationship Management (CRM) system, to be able to ask informed questions.

This demonstration system is trained to understand related to device batteries, and the CRM system stub has hard-coded entries for an iPhone and an iPad for the user.

Please log in to start the demonstration. Usernames will not be validated, and are only for demonstration purposes.

Pre-Rendered Dialog

This is a pre-rendered dialog, which means that $mdDialog doesn't compile its template on each opening.

The Dialog Element is a static element in the DOM, which is just visually hidden.
Once the dialog opens, we just fetch the element from the DOM into our dialog and upon close we restore the element back into its old DOM position.
